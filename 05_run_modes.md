# 05 · 跑法总览 —— 四种积累 + 一次校准

> 给**作者**看的操作地图：现在有几种把诗「照亮」的跑法、各自是什么信号、什么时候用、怎么一句话推进一轮。
> 给 **agent** 看的是每种跑法对应的 skill / 命令；机制细节在 `03_runner_and_coverage.md`（数据结构）和 `theater/runners/DISPATCH-SPEC.md`（派发机制）。
>
> 一句话分工：**盲读铺广度，加厚补薄，跟帖掘深度，点赞筛质量，校准出榜单。**

---

## 一张表看全

| 模式 | 一句话 | 信号 | 进榜单/校准？ | 成本 | 怎么触发 |
|------|--------|------|:---:|------|----------|
| **盲读 blind** | 读者只见一首诗，独立打分 + 短评 | 浅（一次性反应） | ✅ 是 | 线性、可无限累积 | `/dispatch-reads` |
| **加厚 thicken** | 盲读的批量变体：给覆盖薄的每首诗补 N 层盲读 | 浅（同盲读） | ✅ 是（本质就是盲读） | 线性，一次扫全集 | `plan_thicken.py --layers N` |
| **跟帖 thread** | 对已有长评开一场讨论，逐轮接楼、可深达多级 | 深（多轮演化、立场变化） | ❌ 否 | 随楼层膨胀，需封顶 | `/dispatch-thread` |
| **点赞 votes** | 读者给已有评论投 赞/踩/跳过 | 元（评论本身质量） | ❌ 否（另算） | 线性，按评论数 | `/dispatch-votes` |
| **校准 calibrate** | 吃全部盲读，算读者严宽、出榜单 | —（横切汇总，不产新读） | 它就是校准 | 秒级、纯计算 | `python theater/src/calibrate.py` |

**信号为什么分深浅**：盲读/加厚是「一双眼睛看一次」的即时反应，浅但可无限叠加——这是「重读积累」的稳定来源，也是北极星「由世间所有的所见将它命名」的主干。跟帖是「一群眼睛来回看」的演化过程，深但贵。点赞不产生新的「读」，它是对已有读的**再评价**。

**为什么只有盲读进榜单/校准**：榜单要可比，就得所有分来自同一种「独立初见」条件。跟帖不评分（`score` 落 `null`），点赞是元信号，都刻意留在榜单之外——它们是另一路光，不进那本账。（FROZEN：见 `00_START_HERE.md`。）

## 盲读 blind —— 主干，先跑它

- **是什么**：一个读者（模型 × 人设）只看到一首诗，看不到别人的评论、看不到诗的写作背景（除非人设特许 `knows_诠释`），独立给 0–10 分 + 两三句短评。
- **什么时候用**：几乎总是。这是日常，成本线性、免费/冗余额度就能长期跑。
- **怎么跑**：`/dispatch-reads`。它替你算覆盖缺口 → `runner.py plan` 生成任务 → `poem-reader` 轻量子代理并发读 → 质检 → `runner.py collect` 入库。
- **产出落哪**：`results/reads/reads.jsonl`（`context_mode="blind"`）。
- **注意**：一诗一子代理是定死的（保纯度）；每条必记 `model` 与 `transport`，将来能按字段过滤掉拉胯的模型。

## 加厚 thicken —— 盲读的「全集补齐」变体

- **是什么**：和盲读**同一种读**，只是**选样策略**不同。盲读默认「挑覆盖最薄的一批来补」；加厚是「全集扫一遍，给每首覆盖不足的诗都补齐到 N 层」，一轮把广度整体抬一档。
- **什么时候用**：想让**整本集子**的覆盖均匀上一个台阶，而不是零敲碎打补几首时。
- **怎么跑**：`python theater/runners/plan_thicken.py --layers 1 [--exclude zq-0280,...] [--out DIR]`，再走和盲读一样的派发 + collect。
  - 每首按 (诗 × 人设) 覆盖数最少的先取，随机 tiebreak；剔除已够/被 hidden 的。
- **产出落哪**：同盲读（`reads.jsonl`，`blind`）。校准/榜单自动纳入。
- **和盲读的关系（别混淆）**：加厚 = 盲读 + 「全集补齐」的选样。数据无差别，差别只在**这一轮挑哪些 (诗×读者) 去跑**。所以没有单独的 `dispatch-thicken` skill——它就是盲读的一种 plan 方式。

## 跟帖 thread —— 唯一的「深度」通道

- **是什么**：对一首**已有长评**的诗，开一场读者围绕那篇长评的讨论。逐轮指定「接哪一层楼」，读者顺着祖先链往下接话，可深达多级；深楼带「立场是否改变」信号，回复顺带对被接的楼投一票（顺势票，弱信号）。
- **什么时候用**：只对**少数精选、已有值得讨论的长评**的诗定点开。这是「偶尔办的沙龙」，不是日常。
- **怎么跑**：`/dispatch-thread`。底层 `plan_thread.py invite`（派发接楼）→ 子代理写回执 → `plan_thread.py collect`（带引用校验，抄不到原话的楼会被拒）。`void` 隐藏坏楼。**不知道接哪层就 `plan_thread.py nextround`**——按争议度/立场撼动/楼主下场自动挑候选、打印现成命令，只提案不派发。
- **分档（v1.5）**：一楼是**广度档**（fraction 0.5、便宜通道、prompt 不带立场惯性）；2 级起是**深度档**（fraction ~0.25、更强通道、全套机械）。只谈贵贱不点名模型。优先回应权（楼主 + 被回击者）由 invite 自动保障；同层默认不重复邀请。
- **产出落哪**：`reads.jsonl`（`context_mode="thread"`，`score=null`）+ 侧车 `results/threads/`（链深/立场变化/沉默/void）。前端「跟帖」页展示，不进榜单/校准。
- **成本纪律**：祖先链有 token 封顶（`THREAD_TOKEN_BUDGET`），成本随楼层膨胀，务必给层数上限；贵模型只留给深度档和少数精读。

## 点赞 votes —— 给评论质量筛一遍

- **是什么**：让读者对**已有盲读评论**（短评或长评）投 赞（论证咬住了这首诗）/ 踩（分数与理由脱节、对不上诗）/ 跳过（没错但空泛，换首诗照样成立）。
- **什么时候用**：想知道「哪些评论其实没说到点子上」，据此撤评（curation）或看某轮盲读整体质量时。
- **怎么跑**：`/dispatch-votes`。底层 `plan_votes.py invite`（可 `--targets` 指定评论，或 `--poem-ids` 按诗）→ 子代理投票 → `plan_votes.py collect` → `plan_votes.py tally`（`--worst N` 全集撤评总览，已 hidden 的自动跳过）。`void` 作废坏票。
- **产出落哪**：`results/votes/votes.jsonl`（+ `void.json` 侧车，票据永不删）。
- **两种票分得清**：点赞模式发起的是**主动票**（撤评只看它）；跟帖回复顺带的是**顺势票**（`source="piggyback"`，几乎恒为赞、弱信号，前端灰显、tally 分栏）。
- **提示词要点**：投票走**两步漏斗**——先看硬伤判 down，没硬伤默认落 skip，只有抄得出「只对这首诗成立」的原句才挣到 up。（2026-07-18 B 复验证明这让 skip 从死档复活。）
- **加精 best（v1.5）**：批量装箱（默认 ≤8 条/箱、4000 字预算）时回执多一条「同箱最扛得住的一条」。逐条投票是绝对判断、有正向偏置（实测八成 up）；加精是相对判断、不受偏置影响，是白捡的区分度信号。前端 ⭐N 显示，`vote="best"` 落同一份 votes.jsonl，旧统计自动忽略。

## 校准 calibrate —— 横切，不产新读

- **是什么**：吃全部盲读，估每个读者的严宽（有人天生打分高/低），据此出可比的榜单。不产生新的「读」。
- **怎么跑**：`python theater/src/calibrate.py`（服务器长驻时会自动重载最新校准代码）。产出落 `results/calibration/`。
- **注意**：只吃 `context_mode="blind"`，天然不受跟帖/点赞影响。**榜单不得由 LLM 排名**（FROZEN）。

## 典型一轮（作者只需说一句，agent 照做）

1. **补广度** → 「跑一轮盲读」或「加厚一层」：`/dispatch-reads`（薄处优先）或 `plan_thicken.py --layers 1`（全集补齐）。
2. **出榜单** → `calibrate.py`，看新覆盖下榜单怎么动。
3. **筛质量**（可选）→ `/dispatch-votes` 对可疑批次投票，`tally --worst` 看最该撤的，撤评。
4. **掘深度**（偶尔）→ 挑一两首有好长评的诗，`/dispatch-thread` 开一场沙龙。
5. **看现状** → 直接开前端最直观（`python theater/src/server.py` → localhost:8737）；要单看覆盖账用 `python theater/runners/runner.py coverage`（加 `--full` 看每首诗每人设的明细）。
